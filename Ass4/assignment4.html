Hands, Hands, Hands Everywhere - Introduction

The following assignment uses a multidimensional dataset of a set of human hands. Each hand is created by a set of 56 points(x,y), corresponding to 112 dimensional data representation. In order to find out how different the hands are, a principal component analysis can be used. A PCA, takes a high dimensional dataset an represents it in a lower dimensional space, using principal components. Each principal component is orthogonal to the others. In practice one performs an eigenvalue decomposition of the dataset, returning n eigenvalues, and n eigenvectors, where n is the dimensionality of the data. The eigenvectors are then sorted in descending order based on the eignvalues from high to low. An eigenvector corresponds to a principal component.

Since the data consists of 112 dimensions, there are 112 principal components. gives us an ordered list of components where, the first principal component captures the most variance of the data, the second captures the next most and so on. If the data is plotted along the first two principal components, we are able to see how hands differ from eachother, since they are located at very different positions along the two principal components.

Below you will find two plots which provides further insight to how the hands actually look, and how variance in between is captured by PCA. We have used the handed out data, which has an already performed principal component analysis. This data is plotted in a scatterplot to the right. The first principal component is the the x-axis, the second the y-axis. This captures how hands vary along these components. One could have used one of the other principal components and might have found other interesting hand variations in the data. However since we are using the first two principal components we do capture most of the overall variance in the data.

The second plot shows a hand corresponding to the point marked in the scatterplot. With this annotation one has the opportunity to locate and visually see the variance in the data. Take for example two points, which are approximately at the same y-value (the second principal component). Now pick one point with a low x-value (the first principal component) and one with a high x-value. Moving between these points in the scatterplot, reveals hand varianceby drawing different hands in the first plot. Similar to this, the second principal component captures another interesting difference in the hands.

Choices

We have made couple of small decisions in order to let the viewer explore and gain an intuition about the data presented. First, the active transition in the first plot (showing hands), is directly coupled to the point chosen in the scatterplot when one hovers over it. It makes the interaction more smooth, since we don't want the user to click. We also wanted to highlight the difference to the mean hand, therefore when a point is hovered, the original mean hand (in black) is shown underneath the hovered hand. We also highlight both the actual hand in the scatterplot (with a color indicator), while simultaneously updating a legend corresponding to the hand reference in the dataset. This way a user is informed about hands with a large variance. If a user wanted to do something further with the dataset (removing an outlier or similar), they would easily be able to do so.
